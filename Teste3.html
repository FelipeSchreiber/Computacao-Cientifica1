def condicao_parada(f,x,xtol,ytol,step,niter,maxiter):
    return (np.abs(f(x))<ytol) or (np.abs(step)<xtol) or (niter>maxiter)
def condicao_parada_bissecao(a,b,xtol,niter,maxiter,ytol,f):
    return abs(a - b) < xtol or niter == maxiter or np.abs( f((a+b)/2)  ) < ytol
def newton_pts(f, df, x, xtol=1e-8, ytol=1e-8, maxiter=100):
    """
    Mtodo de Newton para encontrar uma raiz de  f  a partir do valor inicial  x.
    Retorna todos os pontos pelos quais o algoritmo passa.
    """
    stop = False
    niter = 0
    pts = []
    pts.append(x)
    while(not stop):
        step = f(x)/df(x)
        x -= step
        stop = condicao_parada(f,x,xtol,ytol,step,niter,maxiter)
        pts.append(x)
        niter += 1
    return pts     

def bissecao_pts(f, a, b, xtol=1e-8, ytol=1e-8, maxiter=100):
    """
    Encontra uma raiz de  f  pelo algoritmo da bissecao, retornando
    todos os pontos pelos quais o algoritmo passa.
    """
    num_iter = 0
    pts = []
    #supoe fb > fa
    if(f(a) > f(b)):
        aux = b
        b = a
        a = aux
    fcalls = 2
    while(not condicao_parada_bissecao(a,b,xtol,num_iter,maxiter,ytol,f)):
        m = (a+b)/2
        pts.append(m)
        if(f(m) > 0):
            b = m
        else:
            a = m   
        num_iter += 1
        fcalls += 1
    pts.append((a+b)/2)
    return pts

def cria_poli(l):
    def poli(x):
        return np.sum([l[i]*(x**i) for i in range(len(l))])
    return poli

def cria_derivada_poli(l):
    def dpoli(x):
        return np.sum([i*l[i]*(x**(i-1)) for i in range(1,len(l))])
    return dpoli

A partir do grfico "Comportamento de P(x) e sua derivada" percebemos que h 4 assntotas para o tamanho do passo, que invertem de sinal, de acordo com o valor da derivada e de P(x). Importante notar que essas assntotas se originam quando a derivada aproximadamente 0 (isto perto dos pontos "-0.062" e "0.927") e como p(x) no pequeno suficiente nesses pontos, o tamanho do passo "explode". No a toa que quando o ponto inicial coincide com essas assntotas o nmero de iteraes necessrias aumentam bastante.

  • Vejamos a 1 assntota quando X inicial, $X_0$, um pouco menor que -0.062. Ela positiva dado que a derivada e P(X) so positivas. Isso faz com que $X_1$ se desvie bastante para a esquerda devido a operao $X_1 = X_0 - step$. Logo, para $X_0$ situado a esquerda de -0.062 o algoritmo converge para a raiz $x = -0.575$.
  • Vejamos a 2 assntota, quando $X_0$ infinitesimalmente maior que -0.062. Ela negativa dado que a derivada negativa e P(x) positivo. Isso faz com que $X_1$ se desvie bastante para a direita. Logo, para $X_0$ situado a direita de -0.062 o algoritmo converge para a raiz $x=1.217$.

    No entanto temos ainda 3 casos possveis, dependendo do quo distante estamos de "-0.062" para a direita:

    1. H um ponto, digamos "a" $a \in (-0.062,0.2)$, em que o tamanho do passo tal que $X_1$ coincide com a 3 assntota, que positiva. Com isso possvel que $X_2$ acabe indo muito para a esquerda e o algoritmo converge para $x=-0.575$ ou at mesmo gere algum ciclo.
    2. Caso em um outro ponto inicial $a+\epsilon$(sendo $\epsilon$ um valor pequeno suficiente) o tamanho do passo for tal que $X_1$ coincide com a 4 assntota, que negativa, ento possvel que $X_2$ acabe indo muito para a direita e o algoritmo converge para $x=1.217$.
    3. Seja ainda $\delta$ um valor suficientemente pequeno tal que $a+\delta > a+\epsilon$ e $a \in (-0.062,0.2)$. Considere ainda que o tamanho do passo em $a+\delta$ seja o suficiente para que $X_1$ caia antes da 3 assntota. Nesse caso o algoritmo converge aproximadamente para $x=0.475$
  • Vejamos a 3 assntota, quando X inicial um pouco menor que 0.927. Ela positiva dado que a derivada e P(x) so ambas negativas. Isso faz com que $X_1$ se desvie bastante para a esquerda devido a operao $X_1 = X_0 - step$. No entanto para determinar para qual raiz ir convergir depender do tamanho do passo nesse ponto, de forma similar como foi discutido para a 2 assntota.
  • Vejamos a 4 assntota quando X inicial, $X_0$, um pouco maior que 0.927. Ela negativa dado que a derivada positiva e P(x) negativa. Isso faz com que $X_1$ se desvie bastante para a direita. Logo, para $X_0$ situado a direita de 0.927 o algoritmo converge para a raiz $x=1.217$.
