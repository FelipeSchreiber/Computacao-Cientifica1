def newton_solve(f, df, y, x0, xtol=1e-8, ytol=1e-8, maxiter=100):
    """Newton's algorithm for solving  f(x) = y  starting from  x0.
    Also returns the number of evaluations of f"""
    nevals = 0
    x_at = x0
    pts.append(x_at)
    for iteracao in range(maxiter):
        passo = (f(x_at)-y) / df(x_at)       
        x_at -= passo
        pts.append(x_at)
        nevals += 1
        if abs(f(x_at)-y) < ytol or abs(passo) < xtol:
            nevals += 1
            return x_at,nevals
    return x_at,nevals

def newton_inv(f, df, ys, x0, xtol=1e-8, ytol=1e-8, maxiter=100):
    """Newton's algorithm for solving  f(x) = y  for each  y  in the list  ys.
    Returns two lists, one of the (approximate) roots, another of the iterations for each one."""
    xs     = []
    nevals = []
    for y in ys:
        if len(xs) != 0: #Se j tivermos valor no cache, vamos us-lo como ponto inicial
            x_at = xs[-1]
        else: #Se for o primeiro, ento usar x0 fornecido
            x_at = x0
        iters = 0
        for iteracoes in range(maxiter):
            passo = (f(x_at)-y) / df(x_at)       
            x_at -= passo
            iters += 1
            if abs(f(x_at)-y) < ytol or abs(passo) < xtol:
                xs.append(x_at)
                nevals.append(iters)
                break #Temos que interromper aqui para adicionar apenas 1 elemento a cada lista para cada y iterado
    return xs, nevals

Essa caixa semelhante s anteriores no sentido em que eventualmente em uma das iteraes o valor de x chega prximo a 0 ou chaga at a ser negativo. A dificulde reside no fato de que para valores de y negativos, temos que eventualmente durante a execuo do algoritmo o valor de x fica entre 0 e 1. Para o caso do log, temos que o valor de x atualizado por $x = x - (\log(x) - y)x = x(1 -\log(x) + y)$. Dessa forma, considerando que o x inicial positivo, o seu sinal s inverte quando $1 - \log(x) + y < 0$. Quando isso ocorre na prxima iterao teremos um erro pois log(x) no est definido para $x \leq 0$. Assim, para evitar isso necessrio que o x inicial seja tal que $\begin{equation} 1 - \log(x) > - y =\\ -log(x) > -y - 1 =\\ x < exp(y+1) \end{equation}$

Explique porque o mtodo de aproximaes funciona.

Student's answer Score: 3.0 / 2.0 (Top)

O mtodo funciona pois a cada y encontramos uma raiz ainda mais prxima que servir de ponto de partida para o prximo y. Seja $x_{n-1}^*$ a raiz encontrada pelo mtodo para $Y_{n-1}$, logo $\log(x_{n-1}^*) \approx Y_{n-1}$. Colocando na equao encontrada anteriormente, temos que a diferena entre $Y_{n-1}$ e $Y_n$ a fim de que o algoritmo no encontre x negativo deve ser tal que: $\begin{equation} -\log(x_{n-1}^*) > - Y_n - 1\\ -Y_{n-1} > -Y_n -1\\ -(Y_n - Y_{n-1}) < 1 \end{equation}$ Como a diferena entre cada $Y_n$ e $Y_{n-1}$ na ordem de 2/3 < 1, temos que a condio satisfeita.
